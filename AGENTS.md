# AGENTS.md

このリポジトリの用途は「改善要望 → 経路アルゴリズム変更 → 画像確認」だけとする。

- コースマップ`data/courses/normalized/`を残す。
- 出力は原則`outputs/result.png`の1枚だけにする。
- 車両・速度・時間・追従・期待モデル、TXTログ、校正、ランキングを追加しない。
- 評価軸は最大オフセット、最小半径、曲率変化率`dκ/ds`とする。
- 一定速度の角加速度は`α = v² dκ/ds`として解釈する。
- LN5.xx_GitHubは読み取り専用。Python候補を実走可能扱いにしない。
- 組み込み候補は最大6100点、単精度、heapなし、追加23 KiB以下、RX651で7秒以内を前提とする。
- コード、コメント、報告は日本語を基本とする。

画像出力の標準:

- 経路線はすべて細線（標準`lw=1.8`）でそろえる。原コースを灰色、現行ファーム経路を青、比較・改善経路を緑で表示する。
- 原コースだけを太線にして、他の経路を見えにくくしない。
- 画像タイトルは「年 + コース名 + 比較内容」の日本語にする。凡例も日本語にする。
- 右側には曲率変化率のグラフを原則置かず、短縮率、最大オフセット、最小半径、判定を表形式で表示する。
- 評価条件（最大オフセット100 mm以下、最小半径60 mm以上）を画像内に明記する。
- 赤い`×`を置く場合は、凡例に「曲率変化の最大位置」または「次の改善対象」と意味を明記する。経路線と混同させない。
- 画像中の説明文、凡例、評価コメントは日本語にする。単位と数式の記号は必要に応じて残す。
- 複数コースを比較するときは`outputs/<用途名>/`へコース名付きPNGを保存し、単一コースの確認時は`outputs/result.png`を使う。

検証:

```powershell
$env:PYTHONPATH="src"
python -m unittest discover -s tests -v
python -m robotrace_shortcut_lab
```
