# 設計

## 残す処理

```text
正規化済みコースTSV
  → 現行Elastic Bandの高速近似
  → 長直線検出
  → 曲率連続の接続
  → offset / radius / dκ/ds評価
  → result.png
```

RX651側では経路生成後に最大`dκ/ds`だけを走査し、固定161点窓の7重対角Cholesky解法を適用する。追加作業RAMは6440byte以下、heapは使わない。候補を書き戻す前後でオフセットと半径を再確認し、改善しない候補は元へ戻す。

現行高速近似は、逐次更新をNumPy同時更新へ置き換えたPC検討用です。2025全日本の旧逐次実装に対してRMS差約1.8 mm、最大差約4.9 mmでした。

## 角加速度

直線の曲率は0、円弧の曲率は`1/R`です。両者を直接接続すると曲率が段差になり、理想モデルでは`dκ/ds`と角加速度が無限大になります。

現在の既定案は、長い直線を優先するため450 mm相当の5次Hermite遷移を置き、位置・一次微分・二次微分を両端で一致させます。遷移を短くして直線コアを延ばしつつ、曲率は連続に保ちます。真のクロソイドは曲率が距離に対して線形になるため将来候補ですが、まずはFresnel積分を持たない小さな実装でピーク評価を回します。

別視点候補として、接続後の最大曲率変化ピークへ固定161点窓の3階差分正則化を反復適用します。これは曲率の急変を直接抑える近似で、クロソイドのようなFresnel積分を必要としません。各反復で最大オフセットと最小半径を再検査し、悪化した候補は採用しません。

60 mmは棄却下限であり、狙う半径ではありません。曲線部の緩和目標は100 mmを維持します。
