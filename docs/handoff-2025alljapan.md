## 7. 経路生成方式

### 7.1 原コース

入力XYをそのまま使う。点数、順序、座標を変更しない。オフセットは全点0 mm。

### 7.2 現行Elastic Band相当

各反復で次を行う。

1. 現在点を元ラインへ戻す
2. 前後点の平均方向へ移動して経路を短く滑らかにする
3. 変位量を75 mm以内へ制限する
4. 7反復ごとに前後20点半径を確認する
5. 半径100 mm未満の周辺を元ライン側へ戻す
6. 始端と終端を元座標へ固定する

最後に最大80回の半径修復を行う。点のindex対応は維持するが、時間選択型と違い、内部状態は法線オフセット1変数ではなくXYである。

### 7.3 時間選択型Frenet経路

新方式では、原コース点`p_i`とその左法線`n_i`に対し、生成点を必ず次で表す。

```text
q_i = p_i + d_i n_i
```

最適化対象はスカラーの横オフセット`d_i`だけである。XY最近傍探索や別区間への再射影は行わない。このため、交差点で近くに存在する別のコース区間へindexが飛ぶことはない。

各反復は固定幅の局所参照だけを使い、次の更新量を足す。

#### 経路長更新

前後点の弦中点へ寄せる移動量を原コース法線へ射影する。

```text
target_i = (q_(i-1) + q_(i+1)) / 2
Δd_length = dot(target_i - q_i, n_i)
```

#### 曲率平滑化更新

4階差分を0へ寄せる固定5点ステンシルを使う。一定曲率を完全に直線化せず、曲率の局所的な凹凸を減らす意図である。

```text
target_i = {4(q_(i-1)+q_(i+1)) - (q_(i-2)+q_(i+2))} / 6
```

#### 曲率変化抑制更新

6階差分を0へ寄せる固定7点ステンシルを使い、`dκ/ds`の局所ピークを抑える。

```text
target_i = {q_(i-3) - 6q_(i-2) + 15q_(i-1)
            +15q_(i+1) - 6q_(i+2) + q_(i+3)} / 20
```

#### その他の制約

- `source_weight × d_i`で元ラインへ弱く戻す
- 1反復の更新量を`step_limit_mm`以内へ制限
- `d_i`を±75 mmへ射影
- 始終端30点は`d_i=0`
- 続く30点はcosine遷移で更新強度を0から1へ上げる
- 12反復ごとに半径不足区間を元ライン側へ戻す
- 最後に最大80回の半径修復

内部配列は現在経路1本分だけであり、候補8本を同時保持しない。

## 9. 幾何評価

### 9.1 半径

現行ファームと同じ考え方で、前後20点と現在点の3点円から半径を求める。ほぼ直線は999999 mmとして扱う。

### 9.2 曲率

前後20点の3点円半径と外積符号から符号付き曲率`κ [1/m]`を求める。

### 9.3 曲率変化率

10 mm点列の量子化ノイズを速度制限へ直接入れないため、曲率を固定21点移動平均してから、実経路距離に対する勾配`dκ/ds [1/m²]`を求める。

この21点平滑はPC専用の重い最適化ではなく、将来Cで固定幅移動平均へ置き換えられる処理である。ただし窓幅の実機妥当性は未確認。

## 10. 簡易速度計画

速度上限は次の最小値で作る。

### 10.1 最大速度

```text
v <= 8.0 m/s
```

### 10.2 GFCP相当

半径100 mmで3.0 m/sになるようにした。

```text
v_GFCP = 3.0 × (radius_mm / 100)^0.33
```

### 10.3 最大角速度

```text
v_omega = deg2rad(1500) / |κ|
```

### 10.4 曲率変化ペナルティ

一定速度近似`α = v² dκ/ds`から次を上限とした。

```text
v_slew = sqrt(300 / |dκ/ds|)
```

300 rad/s²は比較用の仮設定であり、実機の角加速度保証値ではない。

### 10.5 最低・最高速度

上限計算後に1.0～8.0 m/sへ制限する。始端と終端は1.0 m/sに固定する。

### 10.6 加減速スキャン

各区間の実際の点間距離`ds`を使う。固定10 mmではない。

```text
前向き: v_i <= sqrt(v_(i-1)^2 + 2 a ds)
後ろ向き: v_i <= sqrt(v_(i+1)^2 + 2 d ds)
```

前向きと後ろ向きを固定4回繰り返す。`a=10 m/s²`、`d=30 m/s²`。

### 10.7 予測時間

```text
dt_i = 2 ds_i / (v_i + v_(i+1))
total_time = Σ dt_i
```

## 15. 現時点の既知課題

### 15.1 時間選択型がElastic Bandより遅い

時間選択型は経路長では214.7 mm短いが、最大`|dκ/ds|`が約9.6 1/m²大きい。そのため速度ペナルティが強く、予測時間では0.092秒負けている。

現在の更新式は長さ、曲率、曲率変化の幾何的代理量を減らすだけで、予測時間そのものの局所勾配を更新へ使っていない。候補選択だけが予測時間基準である。

### 15.2 曲率変化ペナルティ300 rad/s²は仮値

ログや実機から同定した値ではない。モータ逆起電力、速度別トルク、角速度制御、タイヤ横力、吸引力を含まない。

### 15.3 曲率評価の窓依存

前後20点半径と21点移動平均を使うため、約200～400 mmスケールの形状を主に評価している。10 mm単位の局所折れを過小評価する可能性がある一方、窓を小さくすると入力量子化ノイズで速度プロファイルが細かく振動した。

### 15.4 Elastic BandはCとの完全一致ではない

参照Cは逐次更新、PC版はJacobi型更新である。またPC版は75 mmを強制している。比較基準として「相当」であり、ファームDUMPとの一致検証は未実施。

### 15.5 コースアウトを保証しない

75 mmは暫定値である。実際の許容オフセットは車体投影幅、白線19 mm、競技規定、自己位置誤差、追従誤差を含めて決める必要がある。

### 15.6 追従可能性を評価していない

経路上の曲率・速度だけのオフライン比較であり、Frenet追従誤差、角速度slew、モータ電流、電池電圧、タイヤスリップ、吸引、オドメトリ誤差をシミュレーションしていない。

### 15.7 RX651への移植は未着手

設計上は固定回数、O(N)、局所参照、再帰なし、大規模行列なし、1候補ずつ評価としている。しかしCコード、静的作業配列、実行時間測定、float32差分確認はまだない。

## 16. 次のAIへ推奨する改善順

1. 最初に現在の9テストと`outputs/result.png`を再生成し、基準値を確認する。
2. Elastic Band 6.787秒、時間選択型 6.879秒を比較基準として固定する。
