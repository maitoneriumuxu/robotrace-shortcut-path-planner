# Robotrace Shortcut Path Planner

保存済みのロボトレース大会コース点列から、既存Frenetショートカットと大域ショートカットDAGを生成し、過去LN5.x ATTACK相当の固定速度モデルで予測時間を比較するPC検討環境です。CSV、GUI、実機制御、ファームウェア組み込みは扱いません。

```powershell
$env:PYTHONPATH="src"
python -m unittest discover -s tests -v
python -m robotrace_shortcut_lab --course data/courses/normalized/2025alljapan.tsv --mode reference
python -m robotrace_shortcut_lab --all-courses --mode embedded-lite
```

単一コース結果は`outputs/result.png`、全31大会の回帰結果は`outputs/all_courses.png`へ生成します。

## 大域ショートカットグラフ

原コースの進行indexを持つアンカーを、一定距離、反復なしRDP、曲率極値・符号反転、GFCP低速区間、加減速支配変化、空間的に近い遠方進行区間から自動抽出します。2025年固有indexは使いません。

アンカー間を次の候補で結ぶ有向非巡回グラフを作り、全辺で`start_index < end_index`を保証します。

- 直線
- 入口遷移＋直線＋出口遷移
- 3次Hermite
- 5次Hermite
- 位置・接線・端点曲率を使うG2近似
- 2本のHermite円弧近似を接続する軽量biarc

基準経路に沿う辺も必ず残します。DAGの軽量近似コストで上位K経路を選び、各経路を約10mm間隔へ結合・再サンプリングした後、既存ATTACK速度計画を全経路へ再適用して順位を決めます。辺単体時間の単純加算だけで最終採用しません。

選択後の各点はXY、経路距離、実接触から求めた`source_progress_index`、原コース進行距離、辺ID、意図的白線交差、yaw、曲率、`dκ/ds`、速度を持ちます。将来のVT追従では原コース距離ではなく、この大域経路専用距離indexを使う前提です。

## referenceとembedded-lite

| 項目 | reference | embedded-lite |
|---|---:|---:|
| 目的 | PC上の品質上限確認 | RX651移植候補 |
| アンカー上限 | 256 | 96 |
| 保持候補辺上限 | 20,000 | 1,200 |
| 完全速度再評価 | 上位64 | 上位8 |
| データ構造 | NumPy、通常のPython構造 | 固定配列・heap不要へ置換可能 |

両モードとも乱数、再帰、SciPy汎用最適化、ニューラルネットワーク、大規模密行列を使いません。embedded-liteはfloat32、固定候補数、決定論的な走行前1回計算を意識しています。

## 競技合法性ハードゲート

[NTFロボトレース競技規程](https://www.ntf.or.jp/?page_id=68)のコースアウト条件に合わせ、最終大域探索は「全姿勢でLN5床面投影と幅19mm白線領域の交差が空でない」ことをハード制約にします。板上にあること、経路中心から任意の白線が近いこと、入口と出口だけが白線上にあることでは合格にしません。

白線領域は各コース線分を半幅9.5mmでround cap／round join膨張したポリゴンの和集合です。経路姿勢を並進2mm以下かつyaw変化1deg以下へ固定細分し、各中間姿勢で回転・平行移動した実車多角形との交差をShapelyで検査します。これにより、10mm経路点の間だけ黒床へ離脱するcandidateも不合格になります。板境界がある大会は、同じ全姿勢で実車外形全体が板内かを別のハード制約として検査します。

各姿勢で実際に重なった白線segment集合を求め、その集合だけを状態とする単調DPで`source_progress_index`を選択します。線形補間や`np.maximum.accumulate()`で進行indexを偽装しません。さらに競技条件として、始端から終端まで全LINE segmentを順番に通ることを必須にします。通常のDP進行差は0または1です。元点列が2mmより細かく、同一姿勢で中間segment全てへ実接触している場合だけ複数indexを進められます。未来LINEへ同時接触しても中間segmentが接触集合になければ飛ばせません。白線完全離脱だけでなく、未通過LINEが1区間でもあるcandidateをハード不合格にします。

合法候補には重なり面積、白線の入り込み深さ、接触外形長、完全離脱までの余裕、同時接触ライン数を記録します。`robust`の初期閾値は設定へ分離していますが実機根拠が未確定なので、`legal`とは別の警告値です。浅角交差と長い平行近接はセンサ／VT誤認リスクとして非ゼロのペナルティを持ちますが、完全離脱のハード判定を代用しません。

規定最大直径250mmを円とみなした134.5mmラインチューブは、従来結果を再現する「規定最大車体による非実車・非合法性確認の幾何下限」にだけ残します。reference、embedded-lite、最終採用の合法判定には使いません。

## LN5全車体外形と200mm接触バー

`data/vehicle/LN5_footprint.json`は全車体投影`full_footprint_components_mm`と、白線接触を証明する`contact_witness_components_mm`を分離します。現在のcontact witnessは車体原点を横切る`[-5,+5]×[-100,+100]mm`の横バーです。ユーザー承認済み設計なので`design_confirmed=true`、製作・実測前なので`as_built_confirmed=false`です。

白線接触判定には横バーだけを使います。全車体外形は未確認のため、板外判定には規定最大投影の半径125mm円を保守的に使います。125mm円を白線接触判定へ流用しません。結果は「理論下限」「設計上合法」「実車確認済み合法」の3段階で表示し、現状の合格経路を実車確認済みとは呼びません。

2025年全日本の板境界は、読み取り専用`robotrace_course_cad` commit `523e833...`のJSONから得た板セルを使います。これは白線接触の代わりにはなりません。他大会で境界メタデータがなければ「板境界未確認」と表示します。

## 固定ATTACK速度モデル

参照した読み取り専用LN5.xコミットは`ace1bcb0a16e30032d1317becee519023847e988`と`14afdb2dcd6e7b27dd42a53c80f7adfd16691bd4`です。今回の探索で次の値は変更しません。

| 設定 | 値 |
|---|---:|
| `R10_speed`、最低速度 | 3.6 m/s |
| スタート速度 | ゴール→スタート間の直線助走から算出 |
| ゴール速度 | 自由（3.6m/sへ減速しない） |
| 最高速度 | 13.0 m/s |
| `speed_root` | 0.33 |
| 最小／最大加速度 | 20.0／55.0 m/s² |
| 減速度、`break_kp` | 55.0 m/s²、1.00 |
| omega加速度補間 | 300～1500 deg/s |
| `SEARCH_RAN_SPEED` | 3.6 m/s |
| `max_AALP` | 100（ファーム数値単位） |
| 前後速度スキャン | 固定4反復 |

```text
v_GFCP = 3.6 × abs(radius_mm / 100.0) ^ 0.33
```

1500deg/sは速度ハード上限ではありません。`omega = v × curvature`から、許容縦加速度を300deg/s以下で55m/s²、1500deg/s以上で20m/s²、その間は線形補間します。

AALPは探索速度3.6m/s一定として、`alpha_rad_s2 = 3.6² × dcurvature/ds`を`deg/sの1ms当たり変化量`へ変換し、元実装と同じ`3.6 × sqrt(100 / alpha_firmware)`をGFCPとmin統合します。100をrad/s²やdeg/s²へ読み替えていません。

スタート・ゴール間は入力点列の終点と始点の実座標差から求める。2025年全日本では1.000mである。ロボット中心をゴールライン位置へ静止配置し、この直線を55m/s²上限で助走した到達可能速度を始端上限にする。助走時間は計測へ含めない。直後の減速が必要なら前後スキャンが始端速度をさらに下げる。ゴール通過速度は固定せず、ゴール後の減速も計測時間へ逆伝播させない。

従来2.980秒経路は進行index 1078→2276を5次Hermite 1辺で結び、元コース約11.98mを飛ばします。規定最大直径250mmの仮想円へ基づく近接判定しか通しておらず、LN5実車が移動中に白線へ触れ続ける証拠がないため、reference有効経路と最終結果から除外しました。2.004秒も白線接触制約を緩めた幾何下限であり、採用対象ではありません。

旧3.590809秒経路は横バーがいずれかの白線へ接触し続けていましたが、`157→417`、`834→1199`、`1199→1773`の大域辺で必要な中間LINEをすべて通っていません。ユーザー確認の競技条件に反するため、正式候補から除外しました。

設計legal探索は200mm横バーで継続しています。最終候補は2mm以下・yaw 1deg以下の全中間姿勢で接触集合が空にならず、全LINE segmentを隙間のない実接触DPで網羅するものだけです。取付±2mmとyaw±1degの全方向感度を通った場合をrobustとします。助走境界での初期legalは`157→417`の3次Hermiteと`834→1206`の入口遷移＋直線＋出口遷移で4.376507秒です。完全離脱0、未通過segment 0、最小接触余裕6.758mm、±5mm／±2degまで合格します。

`deep-reference`は2025年の品質上限確認用で、256/384/512アンカー、20,000/50,000/100,000辺上限、詳細legal検査1,200/5,000/20,000辺、K=16/64/256を段階比較します。接線尺度0.20～0.80、左右非対称、法線制御点を含む曲線群と、採用辺周辺のcoordinate descentを使います。助走境界で再探索したlegal最速は`123→431`のG2近似と`802→1206`の5次Hermiteで4.353362秒です。スタート6.981m/s、ゴール8.564m/s、完全離脱0、未通過segment 0、最小接触余裕0.206mmです。位置誤差±1mmで不合格になるためrobustではありません。robust最速は`127→427`のG2近似と`804→1206`の5次Hermiteで4.358689秒、最小余裕4.640mmです。4秒未満は未達です。

## 現在の既知限界

- 全車体床面投影と横バーのas-built取付位置は未実測で、設計legal結果は実車確認済みではない。
- CAD板境界を確認済みなのは現在2025年全日本だけで、他大会は板境界未確認。
- 白線横断時のラインセンサ挙動、VTオドメトリ補正、曲率FF、タイヤ限界、モータDutyを予測時間へ入れていない。
- 幾何下限は白線重なりと曲率変化制約を緩和した楽観値で、有効経路ではない。
- 外形確認後のShapely swept-footprint検査は候補辺ごとに2mm以下へ細分するため、PC referenceで最も重い処理になる。embedded-lite移植では固定格子、符号付き距離場、heading別外形テーブルなどへの等価置換が必要。
- Python予測は実走可能性、実機性能、競技上の安全を保証しない。

速度モデルの由来、AALP単位、計算量、全結果の詳細は`docs/handoff-2025alljapan.md`を参照してください。コースデータの出典とライセンスは`data/courses/README.md`と`data/courses/LICENSE.robotrace_course_cad`にあります。
